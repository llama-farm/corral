Quickstart

Add auth + billing + feature gating to your app in under 10 minutes. We'll use HORIZON (a Next.js military intel dashboard) as the example throughout.

Prerequisites
Node.js 18+, an existing app (Next.js, Express, Hono, etc.), and optionally a Stripe account (test mode works). You don't need Stripe to start ‚Äî free-plan gating works without it.

Step 1 ‚Äî Initialize Corral

Run npx corral init in your project root. The CLI analyzes your project structure, detects your framework and database, and scaffolds everything.

Install the generated dependencies:

bash
npm install @corral/core @corral/next

Step 2 ‚Äî Edit corral.yaml

The generated corral.yaml is a starting point. Customize it for HORIZON's needs ‚Äî add providers, define plans, gate features:

corral.yaml yaml 
app:
  name: HORIZON
  url: https://horizon.mil
  description: Military intelligence dashboard

# Auth providers ‚Äî each one generates login UI + session handling
auth:
  session_ttl: 7d
  providers:
    - google
    - email:
        magic_link: true
        otp: false
    - device  # CLI auth for HORIZON's own CLI

# Billing plans ‚Äî synced to Stripe with `corral stripe push`
plans:
  free:
    price: 0
    description: Basic access. No credit card required.
  pro:
    price: 49
    interval: month
    description: Full analysis suite for analysts.
    trial_days: 14
  enterprise:
    price: custom
    description: Air-gapped deployment + custom SLA.

# Features ‚Äî map to plans, define gate behavior
features:
  basic_search:
    plans: [free, pro, enterprise]
  export_csv:
    plans: [free, pro, enterprise]
  ai_analysis:
    plans: [pro, enterprise]
    gate: blur        # show blurred preview to free users
  real_time_feed:
    plans: [pro, enterprise]
    gate: skeleton    # show skeleton loader to free users
  bulk_export:
    plans: [enterprise]
    gate: block       # hard block with upgrade CTA
  admin_console:
    roles: [admin]      # role-based, not plan-based

# Usage meters ‚Äî track consumption, enforce limits
meters:
  api_calls:
    plans:
      free: 500
      pro: 10000
      enterprise: unlimited
    window: month
    overage: block      # or: charge, warn
  export_rows:
    plans:
      free: 1000
      pro: 100000
    window: month
    overage: warn

# Database ‚Äî Corral shares your app's database
database:
  adapter: prisma
  url: env:DATABASE_URL

# Admin page
admin:
  enabled: true
  path: /admin
  roles: [admin]

Step 3 ‚Äî Run your app

Corral mounts its routes during your app's startup. No separate process needed ‚Äî just run your app as normal:

Step 5 ‚Äî Sync to Stripe

If you added billing plans, push them to Stripe with one command. This creates Products and Prices in Stripe and writes the price IDs back into your YAML (or .env if you prefer):

~/projects/horizon
corral stripe push Using STRIPE_SECRET_KEY (test mode) Created product: HORIZON Pro ‚Üí prod_QxK9mN2aLp Created price: $49/mo ‚Üí price_1OvK9mN2aLp8xQJ Created product: HORIZON Enterprise ‚Üí prod_QxK9mN3bMq Skipped: free plan (no Stripe product needed) Writing price IDs to .env.local‚Ä¶ Done. Run corral validate to confirm.
Test Mode
Corral defaults to Stripe test mode when NODE_ENV !== "production". Use real Stripe test cards (4242 4242 4242 4242) to test the full checkout flow locally.

Step 6 ‚Äî Gate your first feature

Now use Corral's components to gate the AI analysis panel in HORIZON. The ai_analysis feature is configured with gate: blur, so free-plan users see a blurred preview with an upgrade prompt:

src/app/dashboard/page.tsx tsx 
import { FeatureGate } from '@/components/corral/FeatureGate'
import { AIAnalysisPanel } from '@/components/AIAnalysisPanel'

export default function DashboardPage() {
  return (
    <div className="dashboard">
      <h1>Intelligence Overview</h1>

      {/* Free users see a blurred preview + upgrade CTA */}
      <FeatureGate feature="ai_analysis">
        <AIAnalysisPanel />
      </FeatureGate>

      {/* This renders for everyone on free+ */}
      <FeatureGate feature="basic_search">
        <SearchPanel />
      </FeatureGate>
    </div>
  )
}

That's it ‚Äî FeatureGate reads the session server-side, checks the plan, and applies the gate mode from your YAML. No prop drilling, no context setup, no separate API call.

Step 7 ‚Äî Deploy

Corral works with any deployment target that runs Node.js. Set your environment variables and deploy as normal:

Variable Required Description
CORRAL_SECRET Required 32+ char random string for session signing
DATABASE_URL Required Your database connection string
STRIPE_SECRET_KEY Billing Stripe secret key (sk_live_...)
STRIPE_WEBHOOK_SECRET Billing Stripe webhook signing secret
GOOGLE_CLIENT_ID OAuth Google OAuth client ID
GOOGLE_CLIENT_SECRET OAuth Google OAuth client secret
SMTP_URL Email SMTP connection string for magic links
