Agent-Native Design

Every detail built for agents

An agent can add full auth + billing to any app in 9 commands. No screen-scraping. No ambiguity. Just structured I/O.

ðŸ“‹ /api/llms.txt

Auto-generated endpoint on every Corral app. Describes the project's auth config, active plans, feature gates, and API surface in a format LLMs parse cleanly. Follows the llms.txt spec. Also served at corralauth.dev/llms.txt.

ðŸ“– CORRAL.md

Generated at init time and updated on every corral add. A project-specific guide: what's configured, what routes exist, what gates are active, how to test auth locally. Designed for agent system prompts.

âš¡ corral add <thing>

Incremental commands that do one thing well. corral add feature ai_analysis --plans pro --gate blur. Idempotent. Safe to run twice. Agent-friendly JSON output.

ðŸ”¢ --json everywhere

Every CLI command accepts --json and exits with structured output + a numeric exit code. Parse it, branch on it, pass it to the next command. No screen-scraping.
