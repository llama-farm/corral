Agent-Native

For agents, by agents

Every command outputs --json. Every project gets a CORRAL.md and an /api/llms.txt endpoint. An agent can add full auth + billing to any app in 9 commands.

ğŸ“‹ /api/llms.txt

Auto-generated endpoint on every Corral app. Describes the project's auth config, active plans, feature gates, and API surface in a format LLMs parse cleanly. Follows the llms.txt spec.

ğŸ“– CORRAL.md

Generated at init time and updated on every corral add. A project-specific guide: what's configured, what routes exist, what gates are active, how to test auth locally.

âš¡ corral add <thing>

Incremental commands that do one thing well. corral add feature ai_analysis --plans pro,enterprise --gate blur. Idempotent. Safe to run twice. Agent-friendly output.

ğŸ”¢ --json everywhere

Every CLI command accepts --json and exits with structured output + a numeric exit code. Parse it, branch on it, pass it to the next command. No screen-scraping.

The 9-command workflow an agent uses to add auth to HORIZON:

agent workflow
corral analyze --json # 1. understand the project corral init --yes --json # 2. scaffold everything corral add provider google # 3. add OAuth corral add plan pro --price 49 # 4. add paid plan corral add feature ai_analysis --plans pro # 5. gate a feature corral add meter api_calls --limit 10000 # 6. add usage meter corral stripe push --json # 7. sync to Stripe corral validate --json # 8. verify everything corral seed --env test --json # 9. seed test users
